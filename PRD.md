# [PRD] 인테리어 이미지 변환 및 가구 편집 서비스 요구사항 정의서

본 문서는 2026년 6월 29일 진행된 미니 프로젝트 기획 회의록을 바탕으로 작성된 제품 요구사항 정의서(PRD)입니다. 프로젝트 범위에 따라 **이미지 생성 파트**를 중심으로 구체적인 사양을 정의합니다.

---

## 1. 프로젝트 개요
* **제품명**: AI 기반 2D 인테리어 이미지 스타일 변환 및 가구 편집 서비스
* **목표**: 2D 실내 공간 사진과 프롬프트를 입력받아, 공간의 구조를 보존하면서 인테리어 스타일을 변환하고 특정 가구를 지능적으로 편집(교체/스왑)하는 이미지 생성 백엔드 시스템 구축
* **개발 마감 일정**: 목요일 ~ 금요일 내 동작하는 기본 서비스(MVP) 개발 완료

---

## 2. 핵심 기능 요구사항 (이미지 생성 파트)

### 기능 ①: 인테리어 스타일 변환 (2D → 2D)
* **목적**: 원본 실내 공간 사진의 구도와 경계선을 유지한 채 전체적인 스타일(모던, 내추럴 등) 변환
* **입력 사양**: 원본 2D 공간 이미지 + 원하는 인테리어 스타일 텍스트 프롬프트
* **출력 사양**: 구도가 보존되고 스타일이 반영된 완성 2D 인테리어 이미지
* **핵심 기술 및 구조**:
  1. **가이드라인 추출**: 원본 공간 사진에서 선 구조를 감지하기 위해 **M-LSD Hough Line** 추출 노드 사용
  2. **구도 보존 생성**: 추출된 Hough Line 정보를 **ControlNet** 가이드로 활용하여 Stable Diffusion 모델 기반 이미지 렌더링

### 기능 ②: 가구 스타일 변경 (Inpainting)
* **목적**: 이미지 내 특정 가구만 선택하여 주변 배경의 훼손 없이 가구의 스타일/재질 등을 교체
* **입력 사양**: 원본/완성 이미지 + 변경 타겟 가구 클래스명(예: `chair, couch, bed, dining table`) + 교체 스타일 프롬프트
* **출력 사양**: 타겟 가구 영역만 자연스럽게 변경된 완성 이미지
* **핵심 기술 및 구조**:
  1. **자동 영역 세그멘테이션**: YOLOv8 객체 탐지 모델로 가구 Bounding Box를 찾고, SAM(Segment Anything) 모델로 정밀 실루엣 마스크(`SEGS`) 생성
  2. **배경 왜곡 방지 및 경계 융합**: 
     * `ComfyUI-Impact-Pack`의 `SEGSDetailer` 노드를 사용하여 지정된 마스크 영역만 크롭 후 인페인팅 적용하여 원본 배경을 픽셀 단위로 완벽하게 보존
     * `feathering` 파라미터(기본값: `20`) 조절을 통해 교체된 가구와 원본 배경 경계선을 자연스럽게 블렌딩 처리
     * 원본 가구 구조 유지를 위한 ControlNet Inpaint 연동

### 기능 ③: 가구 위치 교환 (위치 스왑)
* **목적**: 완성 이미지 내 두 가구(A, B)의 위치를 서로 맞바꾸는 기능
* **입력 사양**: 완성 이미지 + 교환 타겟 가구 A 및 B 영역 정보
* **출력 사양**: 가구 A와 B의 위치가 서로 바뀐 최종 이미지
* **핵심 기술 및 구조**:
  * **2단계 순차 인페인팅**:
    1. **1단계**: 가구 A 마스크 영역에 가구 B를 배치하는 Inpainting 수행
    2. **2단계**: 가구 B 마스크 영역에 가구 A를 배치하는 Inpainting 수행

## 3. 기술 스택 및 인프라 사양
* **이미지 생성 모델**: Stable Diffusion v1.5 (`v1-5-pruned-emaonly.safetensors`)
* **ControlNet**: ControlNet Inpaint (`control_v11p_sd15_inpaint.safetensors`), M-LSD (인테리어 변환용)
* **객체 탐지 및 세그멘테이션**: YOLOv8x OpenImage v7 (`yolov8x-oiv7.pt`), SAM (`sam_vit_h_4b8939.pth`)
* **커스텀 노드**: comfyui-helper-nodes (Antigravity 자체 노드), ComfyUI-Impact-Pack, ComfyUI-Impact-Subpack, comfyui-segment-anything
* **하드웨어 가속**: NVIDIA GPU 기반 CUDA 가속 활성화 (로컬 VRAM 8GB 이상 준수)
* **처리 방식**: 이미지 생성에 수반되는 모든 무거운 연산은 백엔드(ComfyUI API 및 커스텀 파이프라인)에서 비동기 처리

---

## 4. 제약 및 제외 사항
* **3D 렌더링 배제**: VRAM 용량 부족 등 로컬 실행 제약으로 인해 2D → 3D 변환 모델 로컬 구동은 기능 범위에서 제외함. (외부 유료 API 연동 여부는 추후 필요시 검토하며, 이번 MVP 이미지 생성 파트 개발에서는 배제)
