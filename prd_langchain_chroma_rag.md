# 📚 랭체인(LangChain) & 크로마DB(ChromaDB) 기반 한국어 소설 RAG 프로젝트 PRD

## 1. 프로젝트 개요
- **프로젝트명**: LangChain & ChromaDB Powered Korean Novel RAG System
- **목적**: 웹에서 A4 용지 약 10장 분량(약 15,000자 내외)의 한국어 소설 데이터를 확보한 뒤, **랭체인(LangChain)**을 활용해 텍스트를 분할(청킹)하고, 가벼운 한글 지원 임베딩 모델을 통해 벡터화하여 **크로마DB(ChromaDB)**에 저장합니다. 이후 기존에 세팅된 **구글 Gemini AI**를 LLM으로 연동하여 FastAPI 기반 웹사이트(`/rag`)에서 실시간으로 질문답변을 테스트할 수 있는 업그레이드된 오픈북 RAG 시스템을 구축합니다.

---

## 2. 비전공자 대학교 1학년을 위한 쉬운 개념 설명 (비유)
1. **랭체인 (LangChain)**: 비유하자면, 도서관 사서 선생님이 일을 빠르고 정확하게 할 수 있도록 돕는 **'만능 도구 상자 겸 자동화 로봇'**입니다. 긴 책을 자르고, 검색하고, AI에게 전달하는 모든 복잡한 과정을 버튼 하나로 연결해 줍니다.
2. **청킹 (Chunking)**: A4 용지 10장짜리 긴 소설을 AI가 한눈에 읽을 수 있도록, 문맥이 끊기지 않는 500글자 정도의 **'한입 크기 카드 조각'**으로 나누는 작업입니다.
3. **임베딩 (Embedding)**: 글자 카드들의 의미를 인공지능이 이해할 수 있는 **'숫자 좌표(위치 지도)'**로 번역하는 과정입니다. 의미가 비슷한 카드일수록 지도 위에서 가까운 곳에 모이게 됩니다.
4. **크로마DB (ChromaDB)**: 좌표가 찍힌 수많은 카드들을 영구적으로 안전하게 보관하고, 0.1초 만에 찾아내는 **'최첨단 전자 도서관 서랍장(벡터 데이터베이스)'**입니다.

---

## 3. 핵심 아키텍처 및 요구사항 명세

### ① 데이터 확보 및 전처리
- **대상 데이터**: 한국어 무료 단편소설 (예: 김유정 <동백꽃> 또는 현진건 <운수 좋은 날> 확장판 등, 약 10,000~15,000자 분량의 텍스트 파일 `data/korean_novel.txt` 준비).
- **청킹 도구**: 랭체인의 `RecursiveCharacterTextSplitter` 사용.
- **분할 설정**: `chunk_size=600` (약 3~4문장), `chunk_overlap=50` (문맥 단절 방지를 위해 문장 끝을 약간씩 겹치게 설정).

### ② 임베딩 모델 및 크로마DB 저장
- **임베딩 모델**: 가볍고 한국어 의미 파악에 탁월한 모델 적용 (아래 **'4장 사용자 선택 요청'** 참고).
- **벡터 데이터베이스**: `Chroma`를 사용하여 `data/chroma_db` 폴더에 로컬 영구 저장(`persist`). 서버를 껐다 켜도 재분석 없이 즉시 로딩되도록 구현.

### ③ LLM 엔진 연동 (`llm_test.py` 참고)
- **생성 LLM**: 이미 검증된 구글의 `gemini-2.5-flash` 모델 사용.
- **역할**: 크로마DB에서 검색해 온 상위 3~4개의 소설 카드 조각(Context)을 바탕으로, 팩트에 입각하면서도 비전공자에게 친절하고 재미있는 답변을 생성.

### ④ FastAPI 웹 서버 및 UI 연동
- **백엔드 구조**: 기존 `main.py`의 라우터 구조를 유지하며, 업그레이드된 `ChromaRAGEngine`을 연결.
- **API 엔드포인트**: `POST /api/rag-query` (질문 수신 및 답변/참고구절 반환).
- **웹 페이지**: `http://127.0.0.1:8000/rag` (`templates/rag.html`) 화면에서 실시간 테스트 가능하도록 연동.

## 4. 💡 [확정] 임베딩 모델 선정 결과

사용자님과의 대화를 통해 **선택지 A: 구글 공식 임베딩 (`text-embedding-004`)**을 최종 임베딩 모델로 확정했습니다.
- **확정 이유**: 기존에 설정된 구글 Gemini API 키(`.env`)를 그대로 활용하므로, 컴퓨터 로컬 리소스(메모리/CPU)를 소모하지 않고 초고속으로 속도와 높은 한국어 정확도를 보장합니다.
- **적용 기술**: `langchain-google-genai` 패키지의 `GoogleGenerativeAIEmbeddings`를 활용하여 ChromaDB와 연동합니다.
