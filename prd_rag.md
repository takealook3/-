# 📚 한국어 소설 기반 RAG(검색 증강 생성) 연습 프로젝트 PRD (개정판)

## 1. 프로젝트 개요
- **프로젝트명**: Korean Novel RAG Practice (Gemini Embedding Powered)
- **목적**: 구글 Gemini의 공식 임베딩 모델(`text-embedding-004`)과 생성 모델(`gemini-2.5-flash`)을 결합하여, AI가 알지 못하는 특정 문서(소설 <운수 좋은 날>)의 내용을 기반으로 질문에 답하는 **"오픈북 질문답변 시스템"**을 구축하고 웹(`/rag`)에서 체험할 수 있도록 구현합니다.

---

## 2. 비전공자를 위한 핵심 개념 이해 (비유)
1. **청킹 (Chunking)**: 소설책을 한 번에 다 기억할 수 없으므로, 문맥이 끊기지 않는 500글자 단위로 **"책을 잘라서 카드 묶음으로 만드는 작업"**입니다.
2. **임베딩 (Embedding)**: 구글 인공지능(`text-embedding-004`)이 카드의 의미를 768개의 숫자 좌표로 변환하여, **"의미가 비슷한 카드끼리 가까운 곳에 모이도록 지도에 위치를 찍는 작업"**입니다.
3. **로컬 고속 검색 (Cosine Similarity)**: 사용자가 질문을 던지면, 질문의 좌표와 가장 가까운 소설 카드 Top 3을 1초 만에 찾아내는 수학적 레이더입니다.
4. **RAG 파이프라인**: 찾아낸 3장의 카드를 AI 대학생(Gemini)에게 손에 쥐여주며 *"이 소설 카드를 참고해서 정답을 말해봐!"*라고 시키는 과정입니다.

---

## 3. 핵심 요구사항 및 상세 명세

### ① 데이터 준비 및 청킹
- **대상 데이터**: 현진건의 단편소설 <운수 좋은 날> (`data/lucky_day.txt`)
- **분할 규칙**: 글자 수 500자 기준, 앞뒤 문맥 단절 방지를 위해 50글자씩 겹치게(Overlap) 분할.

### ② 구글 공식 임베딩 및 로컬 캐싱
- **사용 모델**: Google GenAI `text-embedding-004`
- **로컬 캐싱**: 매번 API를 호출하여 기다리지 않도록, 한 번 분석한 임베딩 좌표는 `data/embeddings_cache.json`에 영구 저장하여 초고속 로딩 구현.

### ③ RAG 검색 및 LLM 생성
- **검색 알고리즘**: 코사인 유사도(Cosine Similarity)를 이용한 상위 3개 청크 검색.
- **LLM 모델**: `gemini-2.5-flash`
- **프롬프트 전략**: 찾아낸 소설 원문 구절을 근거로 제시하고, 소설 속 사실에 입각하여 정확하고 재미있는 한국어 답변 생성.

### ④ 웹 UI 및 API 엔드포인트 구현
- **접속 경로**: `/rag` (`templates/rag.html`)
- **API 엔드포인트**: `POST /api/rag-query`
